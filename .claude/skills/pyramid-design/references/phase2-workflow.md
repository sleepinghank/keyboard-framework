> 本文件是 pyramid-design skill 的阶段二详细流程。

## 本阶段启动规则

**读取上下文：**
```
read: docs/plans/<topic>-phase1.md
```

本阶段**只从 phase1.md 读取上下文**，不依赖 Phase 1 的对话历史。
如果文件不存在，停止并提示用户先完成 Phase 1。

## 步骤 4 — 差距分析与方案制定

声明：`【步骤 4/9 — 差距分析与方案制定】`

**从 phase1.md 读取：**
- 需求摘要
- 代码现状结论（缺失能力、需修改模块）

**执行内容：**

1. **差距矩阵**（现状 vs 目标，结论先行）：
   ```
   | 能力 | 现状 | 目标 | 差距 |
   |------|------|------|------|
   | ...  | 缺失 | 需要 | 新增 |
   ```

2. **主要矛盾定位**：从差距矩阵中识别最核心的 1-2 个问题

3. **提出 2-3 个方案**：
   - 推荐方案放在最前，说明原因
   - 每个方案含：核心思路、优点、缺点、实现代价
   - 遵循 YAGNI：去除不必要功能

4. **逐个提问以完善方案**（每次一个问题，等待回答后才问下一个）

> **[CHECKPOINT 4 — 必须暂停]**
>
> 展示差距矩阵和 2-3 个候选方案。
>
> 明确说："步骤 4 完成，方案已提出。请选择方向后进入步骤 5（方案展示）。"
>
> **在此停止，等待用户选定方向后才能继续。**

---

## 步骤 5 — 方案展示与用户确认

声明：`【步骤 5/9 — 方案展示与用户确认】`

### 设计隔离原则

- 每个模块只做一件事，通过明确接口通信
- 能回答：它做什么？如何使用？依赖什么？
- 内部修改不应破坏外部消费者

### 逐节展示（严格顺序，不可跳过）

**节 5.1 — 架构设计**
- 展示系统架构变化（新增/修改的组件，用文本图示）
- 问："架构设计是否符合预期？" → 等待确认

**节 5.2 — 组件/接口设计**
- 展示新增/修改的接口签名和数据结构
- 问："接口设计是否符合预期？" → 等待确认

**节 5.3 — 数据流与错误处理**
- 展示关键路径的数据流
- 列出错误场景和处理策略
- 问："数据流与错误处理是否符合预期？" → 等待确认

**节 5.4 — 测试策略**
- 列出关键测试场景（单元/集成/E2E各取最重要的）
- 问："测试策略是否符合预期？" → 等待确认

**每节规模：**
- 简单变更：3-5 句话
- 中等复杂：100-200 字
- 高复杂：200-300 字，含文本图示

> **[CHECKPOINT 5 — 必须暂停]**
>
> 所有节确认后，明确说："方案各节已确认。是否继续步骤 6（多角度评审）？"
>
> **在此停止，等待用户回复后才能继续。**

---

## 步骤 6 — 方案多角度评审（并行 Agent Team）

声明：`【步骤 6/9 — 方案多角度评审（并行评审中...）】`

### 并行执行策略

5 个维度完全独立，派发 5 个 Agent Team 同时执行，主线程汇总：

**Agent Team 输入：**
- phase1.md（需求和现状）
- 步骤 5 确认的方案设计内容

**各 Agent Team 任务和输出格式：**

| Agent Team | 检查内容 | 输出格式 |
|----------|---------|---------|
| A - 功能完整性 | 需求是否全部覆盖？有无遗漏场景？ | 通过 / 问题列表 |
| B - 技术可行性 | 目标平台可否实现？依赖是否合理？ | 通过 / 问题列表 |
| C - 可维护性 | 代码结构是否清晰？接口是否稳定？ | 通过 / 问题列表 |
| D - 可测试性 | 每个功能单元是否可独立验证？ | 通过 / 问题列表 |
| E - 风险识别 | 潜在风险有哪些？如何规避？ | 风险+规避措施列表 |

**主线程汇总格式：**
```
## 评审报告

| 维度 | 结论 | 关键问题 |
|------|------|---------|
| 功能完整性 | 通过/问题 | ... |
| 技术可行性 | 通过/问题 | ... |
| 可维护性   | 通过/问题 | ... |
| 可测试性   | 通过/问题 | ... |
| 风险识别   | - | ... |

总体结论：通过 / 需修订（返回步骤4）
```

如有问题：明确说明需要返回步骤 4 修订哪些部分。

> **[CHECKPOINT 6 — 阶段二完成，必须暂停]**
>
> 展示评审报告汇总表。
>
> 如评审通过，明确说："阶段二（方案设计）完成。是否进入阶段三（计划输出）？"
>
> **在此停止，等待用户明确确认后才能进入阶段三。**
