# TAG划分：

TAG采用高两bit作为长度标识，如果高两bit都为1则表示还有后续字节。
### 单字节TAG范围 ：0x00 – 0xBF
### 双字节TAG范围 ：0xC000 – 0xFFFF：


## Tag 值的几种类型

为了方便解析 Tag 值，约定一套标准统一的解析方法。

### 数据类型：

**单字节（HEX）**：单字节无符号整数，范围0-255。

**双字节（WORD）**：双字节无符号整数，范围0-65535。

**三字节（DWORD）**：三字节无符号整数，范围0-16777215。

**四字节（QWORD）**：四字节无符号整数，范围0-4294967295。

**百分比（PCT）**：使用0-100的二进制值表示百分比。

**布尔值（BOOL）**：用于表示开关等场景，0=否，1=是，其他值无效。

**浮点数（FLOAT）**：IEEE 754标准的32位浮点数。

**16进制数组（HEX_ARRAY）**：多个16进制数值组成的数组。

**字符串（STRING）**：ASCII字符串。

### 数据长度
**固定长度**：即长度固定，HEX、WORD、DWORD、QWORD、PCT、BOOL、FLOAT、HEX_ARRAY、STRING都采用这种方式。
**动态长度**：HEX_ARRAY跟STRING类型的TAG都需要长度的字段，格式为TAG+长度+TAG值。 动态长度即长度字段是可以动态变化的，当数据长度不超过0xBF时，长度字段为1个字节，当数据长度超过0xBF时，长度字段为2个字节。长度字段的最高2个bit都为1时表示还有后续字节。

## 常用的值解析方式
**高位 bit 指定单位**：即长度固定，但是通过最高的 1-2 个 bit 标识后续数据的单位或者数量级，这样就可以表示很大的数据范围。这种方式适用于当数据量级变大之后对数据有效精度的要求并没有随着量级变大的情况。
**BITMAP**：即位图，用于表示多个开关状态，每个bit表示一个开关状态，0表示关闭，1表示开启。

## TAG类型定义：
TAG类型使用1个字节表示,按位划分为3个部分:
1. 处理方式(bit7~6):
- 0: TAG_HANDLE_NORMAL   - 普通处理方式(直接存储在TAG节点中)
- 1: TAG_HANDLE_VARIABLE - 变量处理方式(直接读写变量)
- 2: TAG_HANDLE_CALLBACK - 回调处理方式(通过回调函数处理)

2. 访问权限(bit5~4):
- 0: TAG_ACCESS_TYPE_READ_ONLY  - 只读权限
- 1: TAG_ACCESS_TYPE_WRITE_ONLY - 只写权限  
- 2: TAG_ACCESS_TYPE_READ_WRITE - 读写权限

3. 值类型(bit3~0):
- 0: TAG_VALUE_TYPE_HEX      - 单字节十六进制值(0-255)
- 1: TAG_VALUE_TYPE_WORD     - 双字节值(0-65535) 
- 2: TAG_VALUE_TYPE_DWORD    - 三字节值(0-16777215)
- 3: TAG_VALUE_TYPE_QWORD    - 四字节值(0-4294967295)
- 4: TAG_VALUE_TYPE_PCT      - 百分比值(0-100)
- 5: TAG_VALUE_TYPE_BOOL     - 布尔值(0-1)
- 6: TAG_VALUE_TYPE_FLOAT    - 浮点数值
- 7: TAG_VALUE_TYPE_HEX_ARRAY - 十六进制数组
- 8: TAG_VALUE_TYPE_STRING   - 字符串类型
