**演讲时长**：30-45分钟
**演讲对象**：运营、人事、财务等非技术岗位员工
**演讲目标**：让员工理解Claude Code的价值，并能够完成安装和基本使用

## 一、开场引入（5分钟）

### 1.1 悬念开场

如果告诉你，一个原本需要3小时的工作，现在只需要3分钟——而且不需要你会编程，你信吗？

今天我要跟大家分享的，就是这样一款工具。

### 1.2 禅道案例对比

先给大家看一个真实的例子。

**场景**：运营部门需要往禅道里批量导入50条任务。

**传统做法**：
1. 打开禅道网站
2. 手动点击"新建任务"
3. 填写任务名称、指派人、截止日期
4. 保存
5. 重复以上步骤50次

**耗时**：大约3小时，而且容易出错。

**用Claude Code的做法**：
1. 打开一个聊天窗口
2. 输入一句话："帮我把这个Excel里的50条任务导入到禅道"
3. 按回车

**耗时**：大约3分钟，而且不会出错。

这不是科幻电影，而是我们公司现在就能做到的事情。

### 1.3 建立期待

今天培训结束后，你也能做到。

## 二、Claude Code 基本介绍（10分钟）

### 2.1 我们已经认识的AI

首先问大家一个问题：你们平时用过AI吗？

我相信很多同事用过DeepSeek、Kimi、ChatGPT这些工具。这些AI确实很厉害，能回答问题、写文章、帮忙出主意。

但是，这些AI有一个共同的限制：它们只能"说"，不能"做"。

### 2.2 一个简单的比喻

让我用一个比喻来解释：

| AI类型 | 比喻 | 举个例子 |
|--------|------|----------|
| DeepSeek/Kimi | 很聪明的顾问 | 你问"怎么整理这个Excel？"，它告诉你步骤，但你得自己操作 |
| Claude Code | 能干的助理 | 你说"帮我把这个Excel整理好"，它直接帮你整理完 |

**总结**：一个只动嘴，一个既动嘴又动手。

### 2.3 为什么要学Claude Code？

可能有同事会问：AI工具这么多，为什么专门学Claude Code？

三个理由：

**第一，它来自一家很厉害的公司**

Claude Code背后的公司叫Anthropic，由OpenAI的核心团队在2021年创立。OpenAI是做什么的？就是做出ChatGPT的那家公司。

简单说：这是一群最早做出顶级AI的人，离开后创建的新公司。

**第二，它是目前世界上最会写代码的AI**

2024年发布的Claude 4模型，被业内评为"世界最佳编程模型"。

但请注意：**今天我们要学的，不是让你写代码，而是让你用这个能力来帮你工作。**

**第三，它在Google工程师中很受欢迎**

根据公开报道，Claude Code已经成为Google工程师在2025年最青睐的AI编码工具。

如果全球顶尖的技术团队都在用，那它一定有独到之处。

### 2.4 什么是Claude Code？

一句话总结：

**Claude Code就是你的数字员工。**

你用日常语言告诉它要做什么，它会帮你：
- 读取和编辑文件
- 批量处理数据
- 操作系统命令
- 调用其他工具

最重要的是：**你不需要懂编程，只需要会用语言表达你的需求。**

## 三、实操演示（20分钟）

现在到了最关键的部分：实际操作。

我会一步步演示，大家跟着做。过程中有四个命令，我准备了口诀帮大家记忆。

### 3.1 第一步：安装和启动

**操作**：

**方案一：一键安装（推荐）**
1. 联系IT部门获取安装包
2. 双击安装包，按"下一步"完成安装
3. 桌面上会出现"Claude Code"图标

**方案二：命令行安装（适合技术人员）**
1. 打开"命令提示符"（按键盘左下角的Windows徽标键+R，输入cmd，回车）
2. 如果你的电脑已经安装了npm，输入以下命令：
   ```
   npm install -g @anthropic/claude-code
   ```

**说明**：大多数同事使用方案一即可，就像安装QQ、微信一样简单。

### 3.2 第二步：配置API Key

**操作**：

1. 启动后，Claude Code会要求你输入API Key
2. 公司已经为大家统一申请了Key，培训结束后请联系IT部门领取
3. 粘贴Key，按回车

**重要提醒**：API Key就像你的账号密码，请勿分享给他人。

### 3.3 第一个命令：/clear（口诀："清空"）

**场景**：你想重新开始一段对话时使用。

**演示**：

```
你: /clear
```

**效果**：清空之前的对话历史，让Claude Code"忘记"刚才说的话，重新开始。

**什么时候用**：
- 对话太长，想重新开始
- 换了一个新任务
- 觉得Claude Code理解偏离了方向

### 3.4 第二个命令：/init（口诀："认路"）

**场景**：当你打开一个新的工作文件夹时，让Claude Code知道里面有什么文件。

**类比**：就像新员工入职第一天，你带他熟悉办公室各个房间在哪里。

**演示**：

```
你: /init
Claude Code: 我正在分析当前目录的项目结构...
Claude Code: 这是一个电商运营项目，包含以下内容：
  - 产品数据文件夹
  - 订单记录文件夹
  - 竞品分析文件
```

**效果**：Claude Code会"认路"，知道你的工作文件夹里有什么，方便后续操作。

**什么时候用**：
- 第一次打开一个工作文件夹
- 项目结构发生了变化

### 3.5 第三个命令：/compact（口诀："整理"）

**场景**：对话太长时，压缩历史对话，保留关键信息。

**演示**：

```
你: /compact
Claude Code: 已将对话历史压缩，保留了以下关键信息：
  - 你正在处理禅道任务导入
  - 数据文件在 tasks.xlsx
  - 还需要完成字段映射工作
```

**效果**：让对话变简洁，同时不丢失重要信息。

**什么时候用**：
- 对话进行了很多轮，感觉有点乱
- 想快速回顾之前讨论的重点

### 3.6 第四个命令：/skill（口诀："大招"）

**场景**：使用预设的专业工作流程。

**说明**：/skill命令比较高级，今天先了解即可，后续有专门课程讲解。

### 3.7 命令速查表

| 命令 | 什么时候用 | 记忆口诀 |
|------|------------|----------|
| /init | 开始新项目时 | "认路" |
| /clear | 想重新开始时 | "清空" |
| /compact | 对话太长时 | "整理" |
| /skill | 使用预设工作流 | "大招" |

## 五、收尾与行动号召（5分钟）

### 5.1 快速回顾

今天我们学习了：

**一句话总结**：Claude Code是你的数字员工，用日常语言就能指挥它帮你完成工作。

**三个核心命令**：
- `/init` — "认路"，让它了解项目结构
- `/clear` — "清空"，重新开始对话
- `/compact` — "整理"，压缩冗长的对话

### 5.2 你的工作场景

想一想，在你的工作中，有没有这些场景：

**运营同事**：
- 批量修改产品描述
- 从多个来源汇总销售数据
- 自动生成日报、周报格式

**人事同事**：
- 批量处理员工信息
- 整理面试记录
- 生成标准化的通知文件

**财务同事**：
- 对账单数据进行格式转换
- 批量生成报销单模板
- 提取财务报表中的特定数据

**通用场景**：
- 大量重复性的文件操作
- 需要保持格式统一的工作
- 从一个系统迁移数据到另一个系统

这些场景，Claude Code都能帮你。

### 5.3 行动步骤

培训结束后，请做三件事：

**第一步**：安装Claude Code

- 打开命令提示符
- 输入 `npm install -g @anthropic/claude-code`
- 输入 `claude-code` 启动

**第二步**：试着完成一个小任务

- 选一个简单的、重复性的工作
- 用日常语言告诉Claude Code你要做什么
- 看它怎么帮你完成

**记住**：不需要会编程，只需要会说话。

### 5.4 结束语

我知道，面对新工具，大家可能有两个担心：

**担心一**："AI会不会取代我的工作？"

我的答案是：AI不会取代你，它会成为你的得力助手。

就像30年前大家学用Excel一样，现在会用电脑的员工工作效率更高。Claude Code也是一样——它只是一个帮你更高效工作的工具。

把Claude Code当成你的助手，它帮你处理繁琐、重复的工作，让你有更多时间思考、创造、做更有价值的事情。

**担心二**："我学不会怎么办？"

我的答案是：今天学会三个命令就够了。

剩下的，边用边学。遇到问题，问IT部门，或者直接问Claude Code——它自己就是最好的老师。

最后送大家一句话：

**AI的目标不是替代人类，而是解放人类。**

让机器做机器擅长的事，让人做人擅长的事。

## 附录一：故障排除小贴士

**安装失败怎么办？**
- 可能是电脑权限问题，请联系IT部门
- 尝试以管理员身份运行安装程序

**启动报错怎么办？**
- 先检查网络是否正常
- 确认API Key是否正确粘贴（注意不要多复制空格）
- 尝试重新启动Claude Code

**Claude Code操作错了怎么办？**
- 不要担心，它可以撤销操作
- 使用 /clear 重新开始
- 重要文件建议先备份

## 附录二：常见问题

**Q：我的工作是XXX，能用Claude Code吗？**
A：如果你的工作涉及重复性的文件操作、数据处理、格式转换等，Claude Code都能帮上忙。培训后可以找IT同事具体咨询。

**Q：万一Claude Code弄错了我的文件怎么办？**
A：Claude Code在执行操作前会先告诉你它打算做什么，你有确认的机会。建议重要文件先备份。

**Q：我完全不懂电脑，能学会吗？**
A：今天学的三个命令（/init、/clear、/compact）配合口诀，就像学用手机一样简单。有问题随时问IT部门。
