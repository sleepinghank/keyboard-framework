# CH584 BLE 联调实施计划（执行版）

日期：2026-03-05  
关联设计：`docs/plans/2026-03-05-ch584-ble-debug-design.md`

## 0. 说明

- 本计划只包含实施步骤与验收，不包含代码实现。
- 按阶段推进：每阶段完成后必须编译 + 烧录 + 串口验收，再进入下一阶段。
- 蓝牙驱动只上报能力/事件；策略由 middleware 决策。
- 临时日志仅用于联调，最终阶段必须全部清理。

## 1. 阶段A：初始化职责迁移

### 1.1 任务
- [ ] A1. 从 `hal/platforms/ch584/_system_hal.c` 移除 BLE 专项初始化调用（仅保留系统必需初始化）。
- [ ] A2. 在 `application/system/system_init.c` 的 `system_init_drivers()` 接入 `bt_driver_init(false)`。
- [ ] A3. 在 CH584 蓝牙驱动初始化中增加重复初始化保护。
- [ ] A4. 保证改动不影响非蓝牙模块初始化顺序。

### 1.2 验收
- [ ] A-Compile 编译通过。
- [ ] A-Flash 烧录成功。
- [ ] A-Log 串口确认 BLE 初始化只执行一次，无重复拉栈。
- [ ] A-Run 上电后系统稳定，无死循环/异常复位。

## 2. 阶段B：驱动事件上报标准化

### 2.1 任务
- [ ] B1. 复用现有无线事件定义，映射 CH584 驱动关键状态事件。
- [ ] B2. 驱动仅保留接口能力：广播/停止广播/回连/断连 + 事件上报。
- [ ] B3. 删除或下沉驱动中的业务策略分支（时长、是否休眠等）。

### 2.2 验收
- [ ] B-Compile 编译通过。
- [ ] B-Flash 烧录成功。
- [ ] B-Log 串口可观测关键事件序列：初始化完成、广播开始、连接、断联、广播结束/超时。

## 3. 阶段C：Middleware 策略状态机落地（复用 wt_state_t）

### 3.1 任务
- [ ] C1. 在 `middleware/communication/wireless.c` 复用 `wt_state_t` 实现策略状态迁移。
- [ ] C2. 业务规则：上电/唤醒/断联 -> 回连最近主机 + 广播窗口60s。
- [ ] C3. 窗口超时未连接 -> `system_hal_enter_sleep(SYSTEM_POWER_MODE_DEEP_SLEEP, SYSTEM_WAKEUP_GPIO)`。
- [ ] C4. 唤醒后重入恢复流程。
- [ ] C5. 竞态处理：超时触发前检查是否已连接。

### 3.2 验收
- [ ] C-Compile 编译通过。
- [ ] C-Flash 烧录成功。
- [ ] C-Log 串口确认 60s 广播窗口、超时进入深度休眠。
- [ ] C-Wakeup 按键唤醒后自动重试流程生效。

## 4. 阶段D：最近主机掉电记忆

### 4.1 任务
- [ ] D1. 读取持久化“最近主机”作为回连目标。
- [ ] D2. 仅在连接成功后更新持久化。
- [ ] D3. 异常值处理：回退默认槽位，不阻塞广播流程。

### 4.2 验收
- [ ] D-Compile 编译通过。
- [ ] D-Flash 烧录成功。
- [ ] D-Log 串口确认重启后读取最近主机并执行对应回连。
- [ ] D-Reboot 多次重启行为一致。

## 5. 阶段E：连接与 HID 上报可用性

### 5.1 任务
- [ ] E1. 确保仅在 `WT_CONNECTED` 允许 HID 无线上报。
- [ ] E2. 连接后验证键盘报告路径可达。
- [ ] E3. 断联后重连再次验证 HID 上报。

### 5.2 验收
- [ ] E-Compile 编译通过。
- [ ] E-Flash 烧录成功。
- [ ] E-Conn 手机或PC连接成功。
- [ ] E-HID 主机可接收键盘输入（至少验证字母键、修饰键组合）。

## 6. 阶段F：临时日志清理与全链路回归

### 6.1 任务
- [ ] F1. 删除联调新增的全部临时日志代码。
- [ ] F2. 回归验证全链路：广播、连接、回连、断联、重广播、休眠、唤醒、HID。
- [ ] F3. 仅保留必要的正式日志和错误处理。

### 6.2 验收
- [ ] F-Compile 编译通过。
- [ ] F-Flash 烧录成功。
- [ ] F-Regress 全链路通过。
- [ ] F-Clean 无临时日志残留。

## 7. 每阶段执行模板

1. 修改目标文件（仅本阶段范围）。
2. 本地编译。
3. 你烧录并抓串口日志。
4. 依据日志判定通过或补丁修正。
5. 通过后冻结阶段结果并进入下一阶段。

## 8. 风险与回退

- 风险1：CH584 驱动历史逻辑较重，改动易引入状态抖动。  
  回退：每阶段最小提交粒度（由你手动提交），必要时按阶段回滚。

- 风险2：广播超时与连接事件竞态。  
  回退：在 middleware 超时分支增加连接态二次判定。

- 风险3：休眠入口受平台唤醒配置影响。  
  回退：优先保留 GPIO 唤醒单路径，避免同时引入定时唤醒复杂性。
